Commit your code

You've built something real. Now save it properly. Wingman has built-in Gitea integration — your code gets versioned and backed up in one click.

Head to your app card in Wingman and hit the commit button. Your changes are saved to your own git repository on the server.

  1. Open your app card in Wingman
  2. Click the Git tab
  3. Write a short commit message describing what you built
  4. Hit Commit — your code is now saved!

Deploy!

Your app is running locally. Let's put it on the internet so anyone can use it. We'll update the CapRover config and deploy from your app card.

Update CapRover definition
Review the captain-definition file in the project root. Ensure the Dockerfile or image reference is correct for a production build. If there isn't a captain-definition file, create one that uses a Dockerfile with a multi-stage Node build that serves the app on port 3000.

Once committed, head to your app card in Wingman and hit Deploy. Your app will be live in seconds.
