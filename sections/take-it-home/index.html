Commit your code

You've built something real, now take it home. With Wingman's built-in Gitea integration — your code gets versioned and backed up in one click.

  1. From your Edit with AI session
  2. Click cmd -> gitea
  3. Click setup
  4. Click Commit and Push all!
  5. Click Go To Repo

Deploy!

Your app is running locally. Let's put it on the internet so anyone can use it. We'll update the CapRover config and deploy from your app card.

Update CapRover definition
Review the captain-definition file in the project root. Ensure the Dockerfile or image reference is correct for a production build. If there isn't a captain-definition file, create one that uses a Dockerfile with a multi-stage Node build that serves the app correctly.

Once committed, head to your app card in Wingman and hit Deploy. Your app will be live in seconds.
